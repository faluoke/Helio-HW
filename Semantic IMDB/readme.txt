Why should we structure HTML semantically for something like a login form? 
    This helps the search engine parse the data, and helps password managers to enter in data, HTML will add validation if input is semantically friendly
What are some benefits of everyone using similar semantic HTML? 
    Clearer code and easily maintainable, better SEO and accessibility.
What are some disadvantages? 
    If code is already estabilished, have to spend extra time fixing the sementic sometimes having to end up rewriting lots of code.
